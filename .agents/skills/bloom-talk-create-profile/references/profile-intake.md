# Bloom Talk 建档访谈提纲

本文件用于指导“建档时怎么和人说话”，不是字段模板。

目标只有一个：让运行时像人类建档问诊一样，一问一答地把必要信息收集起来，再去写 `profile.md`。

## 交互原则

- 先问，再写
- 每轮只问 1 到 3 个问题
- 先问核心信息，再问补充信息
- 语气自然，不要播报内部读文件过程
- 如果用户已经表达情绪，先顺着情境进入建档，不要机械切换成脚本语言

## 推荐顺序

### 第一轮：先拿到身份和日期基准

优先问：

- 怎么称呼你比较合适？
- 现在是已经确认怀孕、正在备孕，还是需要先澄清当前状态？
- 关于日期基准，你们目前最确定的是哪一个：末次月经、预产期，还是早孕超声结论？

这一轮的目标是先把“在给谁建档”和“日期基准从哪里开始”弄清楚。

### 第二轮：补妊娠背景

可以继续问：

- 这是第一次怀孕吗？
- 之前有没有需要记住的妊娠、流产、分娩或终止妊娠情况？

### 第三轮：补健康背景

可以继续问：

- 有没有需要长期记住的身体疾病、心理健康情况、过敏或禁忌？
- 现在有没有在用的药物或补充剂？

### 第四轮：补当前照护

可以继续问：

- 现在有没有已经联系的医生、助产士或医院？
- 有没有已经做过的重要检查或超声？

### 第五轮：补支持与生活信息

如果用户愿意继续，再问：

- 平时主要是谁在支持你？
- 有没有工作或日常上的限制，需要长期记住？
- 记录上有没有特别偏好？

## 什么时候可以开始写文件

满足以下条件时，就可以先写第一版 `profile.md`：

- 已知道怎么称呼记录对象
- 已知道当前状态
- 已拿到一部分可靠的日期基准
- 已知道是否首孕或至少知道这项暂时未知
- 已问过重大健康背景

这时不需要把所有细节一次问完。

## 写文件前的表达方式

在真正写入之前，可以用一句自然的话过渡，例如：

- “我先根据你刚才给的信息建第一版档案，剩下的细节后面再慢慢补。”
- “目前核心基准已经够了，我先把第一版 profile 建起来。”

不要用这种表达：

- “我现在开始读取文件并落盘。”
- “我将把未知项批量写成未知。”
- “接下来同步多个 slot。”

## 关于未知值

- 只有在用户明确说“不知道”“记不清”“之后再补”时，才把对应字段写为 `未知`
- 不要因为当前还没问到，就默认写 `未知`
